### 参数
> - 使用参数 -d(true/false) true 表示解压 false 表示压缩  
> - -source=file 被压缩或者是被解压的文件名
> - -dest=file 压缩后或者是解压后的文件名

### 2018-06-11  demo版

> - 把huffman算法基本完成了，但是压缩了不是很高，简单看下问题，是因为序列化huffman树的占用了很大的空间 这里有很大的优化空间（毕竟这只是算法的一部分）
